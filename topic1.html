The Evolution of HTTP

Origins at CERN (1989–1991)

Sir Tim Berners-Lee, working at CERN in 1989, envisioned a simple protocol to retrieve and display documents on different machines. By late 1990, he had implemented the first web server, browser, and a protocol called HTTP, and by 1991 shipped the world’s first web site.

HTTP 0.9 (1991)

HTTP 0.9 was a bare-bones protocol: it supported only a single GET command, no headers, and transferred plain text. Despite its simplicity, it proved the concept of hypertext on the internet, enabling users to traverse linked documents easily.

HTTP 1.0 (1996)

As adoption soared, HTTP 1.0 introduced request and response headers, status codes (e.g., 404 Not Found), and the ability to handle images, scripts, and styles. This made the web more interactive and reliable, laying the groundwork for dynamic pages.

HTTP 1.1 (1999)

Key advances in HTTP 1.1 included:

  • Persistent Connections: multiple requests over a single TCP connection.
  • Chunked Transfer Encoding: streaming large files in smaller chunks.
  • Cache Control: fine-grained caching directives for better performance.
  • Virtual Hosting: hosting multiple domains on one IP address.

HTTP/2 (2015)

HTTP/2 tackled performance bottlenecks by adding:

  • Multiplexing: parallel streams over one connection.
  • Header Compression: reduced overhead on repeated headers.
  • Server Push: proactively sending resources to the client.

HTTP/3 (2020)

Built on Google’s QUIC protocol over UDP, HTTP/3 offers:

  • Reduced latency due to faster handshake and connection migration.
  • Mandatory encryption for all traffic, improving privacy.
  • Better resiliency on mobile and lossy networks.

Evolution Summary

Version Year Main Improvement
0.9 1991 Basic GET, plain-text only
1.0 1996 Headers, status codes, media support
1.1 1999 Persistent connections, caching, virtual hosts
2 2015 Multiplexing, compression, server push
3 2020 QUIC, encryption, low latency
